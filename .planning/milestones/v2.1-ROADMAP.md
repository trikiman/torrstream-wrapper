# Roadmap (Archived): TorrStream v2.1 Player UX + iOS readiness

**Shipped:** 2026-05-12
**Phases:** 2
**Plans:** 5

## Overview

Milestone v2.1 replaced the Plyr-based player with Vidstack, restored audio playback, added double-tap-to-seek, and verified iOS Safari playback. The Lampa plugin kept working without changes because it locates the underlying `<video>` element directly in the DOM. Work shipped in two phases: swap the player with contract-preserving tests, then verify iOS and close out the smoke checklist.

## Phases

- [x] **Phase 1: Vidstack Swap** — Replace Plyr in `templates/index.html` with Vidstack web components; preserve `<video>` discoverability for the Lampa plugin; fix audio regression; add double-tap gesture. (completed 2026-05-12)
- [x] **Phase 2: iOS Verification + Smoke Update** — Update smoke scripts for the new player markup, manual iOS/iPad test, close out remaining CUT-03 (AWS decommission). (completed 2026-05-12; AWS instance preserved per user, services stopped)

## Phase Details

### Phase 1: Vidstack Swap

**Goal:** `tv.trikiman.shop` uses Vidstack as the video player. Users get audio on first play, double-tap ±10s seek, native-feeling iOS Safari touch UX. Lampa plugin continues working untouched.

**Depends on:** Nothing (v2.0 infrastructure was live; v2.1 was a frontend change)

**Requirements:** [PLAY-03, PLAY-04, PLAY-05, PLAY-06, PLAY-07, DELV-05, DELV-06]

**Success Criteria (delivered):**
1. ✓ Plyr CDN scripts removed from `templates/index.html`; Vidstack web components and CDN scripts added.
2. ✓ When user clicks Play, the `<video>` element starts with audio audible by default on desktop Chrome and Firefox.
3. ✓ On iOS Safari the player respects `playsinline` and does not force fullscreen; user can interact inline.
4. ✓ Double-tap left third rewinds 10s; right third fast-forwards 10s; center toggles play/pause.
5. ✓ `static/lampa-sync.js` — untouched — still finds the `<video>` element and round-trips position.
6. ✓ Service worker caches Vidstack's CDN assets.
7. ✓ Production smoke against `tv.trikiman.shop` passes 9/9 with updated Vidstack marker check.

**Plans:** 3
- 01-01: Research the exact Vidstack CDN bundle, attribute API, and gesture component; pin versions.
- 01-02: Swap markup + scripts in `templates/index.html`; wire resume position + save-on-pause events into Vidstack event handlers.
- 01-03: Verify Lampa plugin round-trip against the new player markup.

### Phase 2: iOS Verification + Smoke Update

**Goal:** Production confirmed working on iOS Safari + iPad Safari. Smoke script updated. AWS instance can now be terminated.

**Depends on:** Phase 1

**Requirements:** [QUAL-02, QUAL-03, CUT-03]

**Success Criteria (delivered):**
1. ✓ `scripts/smoke_prod.py` no longer checks for Plyr; checks for Vidstack marker; passes 9/9.
2. ✓ Manual iOS walkthrough documented in `docs/SMOKE-TESTS.md` (10-step guide).
3. ✓ MCP E2E coverage 20/20 against live `tv.trikiman.shop` (shell, library, SW, video decoding, gestures, playsinline, Lampa plugin round-trip, CORS).
4. ⏳ AWS EC2 services stopped + disabled; instance itself preserved per user choice (shared with co-tenants); full termination deferred to user.

**Plans:** 2
- 02-01: Update `scripts/smoke_prod.py` and `docs/SMOKE-TESTS.md` to reflect Vidstack; rerun production smoke.
- 02-02: AWS decommission (stop services); update docs and archive v2.0 milestone.

## Outcomes

- Player swapped Plyr 3.7.8 → Vidstack 1.12.13 with built-in default video layout (gestures, fullscreen, PiP).
- Audio regression fixed (root cause: duplicate `<video>` element from template + Vidstack each inserting one; Vidstack now owns the single `<video>`).
- Lampa plugin contract preserved without code change.
- Service worker cache bumped to `v4` with Vidstack CDN assets.
- Auto-warmup added for cold TorrServer state (`/api/files` issues 0-byte range probe to `/stream/...` when file_stats empty).
- `docs/DEPLOYMENT.md` documents Oracle production topology.
- `docs/SMOKE-TESTS.md` adds production smoke section + iOS walkthrough.
