# Requirements (Archived): TorrStream v2.2 Robustness + Coverage

**Defined:** 2026-05-14
**Shipped:** 2026-05-14

Driven by the 2026-05-14 E2E feature audit (8 captured todos in
`.planning/todos/completed/`). All 9 requirements shipped.

## API hygiene

- [x] **API-01**: `/api/files/<hash>`, `/api/position/<hash>` (GET), `/api/remove/<hash>` return **404** when the hash is unknown to both `positions.json` AND TorrServer's library, with the existing diagnostics body preserved.
- [x] **API-02**: All hash-bearing routes reject hashes that don't match `^[0-9a-fA-F]{40}$|^[0-9a-fA-F]{64}$` with **400 invalid hash**. Hashes are normalized to lowercase before persistence and lookup.
- [x] **API-03**: `POST /api/position/<hash>` rejects malformed JSON with **400 invalid JSON**, and missing/non-numeric `position` with **400 missing position** / **400 invalid position payload**, while preserving CORS headers on error responses.
- [x] **API-04**: `GET /static/*` exposes `Access-Control-Allow-Origin: *` so a Lampa-side `fetch('https://tv.trikiman.shop/static/lampa-sync.js', { mode: 'cors' })` succeeds.

## UX completeness

- [x] **UX-01**: Multi-file torrents already opened a file picker (Episode Panel — discovered to be pre-existing during this milestone; the original audit grep used wrong selectors). Single-file torrents preserve straight-to-player UX. Verified end-to-end via browser MCP.
- [x] **UX-02**: Each row in the file-picker has a **⬇ Скачать** button using `<a download="...">` against `/api/stream/...`. Player overlay header gains a download button visible whenever a file is loaded. iOS Safari fallback opens in a new tab + toast pointing at "Поделиться → Сохранить в Файлы". Verified live: anchor `download="Matrix.1999.BDRip.avi"`, server returns 206 with `Content-Range: bytes 0-15/1571913728`.
- [x] **UX-03**: Theme toggle full background color change reaches its target value within **≤16ms** wall-clock from click (measured at frame 0 post-deploy). Implementation: `applyTheme()` writes `document.body.style.backgroundColor` directly, bypassing Chrome's ~680ms style cascade recomputation latency on this DOM (vidstack web components + shadow trees). Original audit baseline: 1432ms.

## Quality

- [x] **QUAL-04**: Every check in the 2026-05-14 E2E feature audit is now a runnable test under `tests/api/` (57 tests via Flask test client + mocked TorrServer) and `tests/integration/` (10 read-only checks against `tv.trikiman.shop`). Tests are safe to run against prod (no mutations in default markers; `e2e` opt-in for full round-trip).
- [x] **QUAL-05**: `.github/workflows/tests.yml` runs `pytest -m smoke` on every PR + push to main, and `pytest -m integration` nightly + on push to main. `scripts/smoke_prod.py` retained as a thin pre-flight; pytest is the source of truth.

## Outcomes

- Theme transition: 1432ms → ≤16ms (89× faster).
- 4 API endpoints gain proper HTTP semantics (404 / 400 instead of swallowing all failures with 200+ok:false).
- Cross-origin fetch of `/static/*` now works (was `TypeError: Failed to fetch`).
- Test coverage: 0 → 67 tests across API contract + live integration. CI hook in place.
