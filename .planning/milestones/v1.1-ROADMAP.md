# Roadmap: TorrStream v1.1 (Cross-Client Position Sync)

## Overview

v1.0 shipped wrapper-internal resume. v1.1 extends resume across clients: a Lampa plugin posts playback time back to the wrapper, the wrapper exposes externally-watched torrents in the continue-row, and re-opening a movie in Lampa picks up where the user left off on any device.

## Phase Details

### Phase 1: Cross-Client Position Sync
**Goal**: Make watch progress follow the user from Lampa to the wrapper (and back) without depending on Lampa.cloud or any third-party sync service.
**Depends on**: v1.0 SYNC-01 (per-file position storage already in place).
**Requirements**: [SYNC-03, SYNC-04, SYNC-05, DELV-03, DELV-04, QUAL-02]
**Success Criteria** (what must be TRUE):
  1. The wrapper homepage "Продолжить просмотр" row includes torrents the user has only watched in Lampa, with a `viewed_in_torrserver` flag distinguishing them from wrapper-local progress.
  2. A Lampa-side plugin POSTs `{position, duration, file_index}` to the wrapper at a regular cadence during playback and on player teardown.
  3. Opening the same torrent in Lampa from a fresh device or browser seeks the player to the wrapper-stored position.
  4. CORS preflight on `/api/position/*` returns the headers Lampa's plugin needs to call the wrapper from `https://lampa.mx`.
**Plans**: 2 plans

Plans:
- [x] 01-01: Wrapper-side visibility for externally-watched torrents and the Lampa plugin v1 (PR #3).
- [x] 01-02: Plugin lifecycle hardening from the post-merge Devin Review finding (PR #4).
