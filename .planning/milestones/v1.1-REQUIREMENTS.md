# Requirements: TorrStream — v1.1

**Defined:** 2026-04-29
**Theme:** Make watch progress follow the user across playback clients, not just the wrapper's own player.

## v1.1 Requirements

### Sync

- [x] **SYNC-03**: Wrapper surfaces torrents that have been watched in TorrServer-backed external clients (e.g. Lampa) in the homepage "continue" row, even when the wrapper has no local position record.
- [x] **SYNC-04**: A Lampa-side plugin posts current playback time and duration back to the wrapper so resume position is shared between the wrapper player and Lampa.
- [x] **SYNC-05**: Re-opening a torrent in Lampa restores playback at the wrapper-stored position when one exists.

### Delivery

- [x] **DELV-03**: The wrapper hosts the Lampa plugin under `/static/lampa-sync.js` so Lampa's standard plugin installer can pull it directly.
- [x] **DELV-04**: `/api/position/*` accepts cross-origin reads/writes so the plugin can call the wrapper from `https://lampa.mx`.

### Quality

- [x] **QUAL-02**: Lampa-side plugin lifecycle listeners (visibilitychange, beforeunload, pagehide) are registered exactly once and the resume seek bails on torrent switch.
