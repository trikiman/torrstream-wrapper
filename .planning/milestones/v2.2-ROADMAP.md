## Phase Details

### Phase 1: API hygiene

**Goal:** TorrStream's HTTP API behaves correctly for every documented edge case.

**Plans:**
- 01-01: Hash format validator + 404 for unknown hash on `/api/files`, `/api/position` GET, `/api/remove`. Lowercase normalization migration. (commit fb22aa8)
- 01-02: Reject malformed JSON on `POST /api/position` (400 + missing-field validation) + `Access-Control-Allow-Origin: *` on `/static/*`. (commit fdd9b10)

**Verification:** All 11 backend regression checks PASS against `tv.trikiman.shop` after auto-deploy. Cross-origin checks from `lampa.mx` PASS.

### Phase 2: UX completeness

**Goal:** Every file in every torrent reachable via UI; theme toggle feels instant.

**Plans:**
- 02-01: Per-file ⬇ Скачать button in Episode Panel rows + player header download button. iOS fallback toast. (commit 415f95e). File-picker modal was discovered to be pre-existing as the Episode Panel (UX-01 verified in place).
- 02-02: Theme transition fix — required four iterations after the initial CSS-only attempts hit Chrome's style cascade recomputation ceiling. Final fix: `applyTheme()` writes `document.body.style.backgroundColor` directly, bypassing the cascade. (commits 44ce659 → b576f5a → b7dbedd → 407494b → fe72be9)

**Verification:** Browser MCP — theme reaches target color in ≤16ms (was 1432ms). Download anchor URL `download="Matrix.1999.BDRip.avi"`, backend returns 206 with correct Content-Range.

### Phase 3: Test harness

**Goal:** Audit's 25 checks become runnable tests; CI catches regressions.

**Plans:**
- 03-01: pytest + Flask test client + mock TorrServer fixture for the smoke layer; `requests` against the live wrapper for the integration layer; pytest.ini markers; GitHub Actions CI hook. (commit 86b4e33)

**Verification:** `pytest -m smoke` → 57 passed in 0.58s. `pytest -m integration` → 10 passed in 7.48s against `tv.trikiman.shop`.
