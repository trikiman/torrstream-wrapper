# Phase 1 Plan 01: Host Preparation (swap, firewall, packages, directories)

**Goal:** Bring `vless-x86-2` (158.101.214.234) from a vanilla Ubuntu 22.04 box to a state where the TorrStream stack can be installed. No app installs in this plan — pure plumbing.

## Tasks

### Task 1.1 — Add swap

Run on the Oracle host:

```bash
if ! swapon --show | grep -q /swapfile; then
  sudo fallocate -l 4G /swapfile
  sudo chmod 600 /swapfile
  sudo mkswap /swapfile
  sudo swapon /swapfile
fi
grep -q "^/swapfile" /etc/fstab || echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
# Prefer RAM over swap for this workload
sudo tee /etc/sysctl.d/99-torrstream.conf <<'EOF'
vm.swappiness=10
vm.vfs_cache_pressure=50
EOF
sudo sysctl --system
```

**Verify:**
- `swapon --show` shows `/swapfile` at 4 GB
- `free -m` shows `Swap: 4095`
- `cat /proc/sys/vm/swappiness` prints `10`

### Task 1.2 — Open 80, 443, 8090 in host nftables

The existing `/etc/iptables/rules.v4`-style config on Oracle Ubuntu uses nftables table `ip filter` with an explicit `REJECT all` at the end of INPUT. We need to add accept rules for the new ports **before** the reject rule and persist them.

```bash
# Insert accept rules BEFORE the final reject. Using iptables-compat keeps the existing rule file happy.
sudo iptables -I INPUT 8 -p tcp --dport 8090 -j ACCEPT
# 80 and 443 are already in the ruleset (ports 6 and 9 from earlier inspection). Confirm:
sudo iptables -L INPUT -n --line-numbers | grep -E 'dpt:(80|443|8090)' | sort -u
# Persist: netfilter-persistent saves both v4 and v6
sudo DEBIAN_FRONTEND=noninteractive apt-get install -y iptables-persistent netfilter-persistent
sudo netfilter-persistent save
```

**Verify:**
- From developer PC: `python -c "import socket; s=socket.socket(); s.settimeout(5); s.connect(('158.101.214.234', 8090)); print('OPEN')"` prints `OPEN`
  - Expected: will fail here because TorrServer isn't installed yet. What we need is TCP connect-time acceptance; run a throwaway listener on 8090 to prove: `sudo nc -l 8090 &` then TCP-connect from PC, expect connect to succeed, then kill nc.
- `sudo iptables-save | grep 8090` shows the rule.
- After reboot (Task 1.4 will test), rules survive.

### Task 1.3 — Install base packages + directories

```bash
sudo DEBIAN_FRONTEND=noninteractive apt-get update
sudo DEBIAN_FRONTEND=noninteractive apt-get install -y \
  python3 python3-venv python3-pip \
  curl unzip rsync \
  debian-keyring debian-archive-keyring apt-transport-https \
  ca-certificates \
  jq

# Caddy official repo (signed) — used by Plan 01-02
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt-get update

# Directories
sudo mkdir -p /opt/torrstream
sudo chown ubuntu:ubuntu /opt/torrstream
sudo mkdir -p /var/lib/torrserver
sudo chown ubuntu:ubuntu /var/lib/torrserver
sudo mkdir -p /var/log/torrstream
sudo chown ubuntu:ubuntu /var/log/torrstream
sudo mkdir -p /etc/torrstream
sudo chown root:root /etc/torrstream
sudo chmod 755 /etc/torrstream
```

**Verify:**
- `python3 --version` reports 3.10 or later
- `apt list --installed 2>/dev/null | grep caddy` is empty (we only added the repo; install in Plan 01-02)
- `ls -ld /opt/torrstream /var/lib/torrserver /var/log/torrstream /etc/torrstream` shows all with expected ownership

### Task 1.4 — Reboot and re-verify

```bash
sudo reboot
# Wait ~60s, then from dev PC:
ssh ... "free -m | grep Swap"     # expect 4095
ssh ... "sudo iptables -L INPUT -n | grep 8090"   # expect ACCEPT
ssh ... "id ubuntu"                # expect ubuntu user still exists
```

## Exit Criteria

- Swap shows 4 GB, persists across reboot
- nftables accepts 80, 443, 8090, persists across reboot
- All base packages installed
- Directory tree at `/opt/torrstream`, `/var/lib/torrserver`, `/etc/torrstream`, `/var/log/torrstream` with correct ownership
- Reboot cycle passes

## Rollback

- `sudo swapoff /swapfile && sudo rm /swapfile && sudo sed -i '/\/swapfile/d' /etc/fstab`
- `sudo iptables -D INPUT -p tcp --dport 8090 -j ACCEPT && sudo netfilter-persistent save`
- `sudo apt-get purge -y iptables-persistent netfilter-persistent jq rsync` (leave python/curl/etc)
- `sudo rm -rf /opt/torrstream /var/lib/torrserver /var/log/torrstream /etc/torrstream`
